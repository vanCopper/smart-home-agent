# Smart Home Agent 项目总结

## 项目概述

基于 OpenClaw + Mac mini 构建一个家庭智能 Agent 系统，具备语音交互、多设备控制、多成员协调、长期学习能力，通过 iPad 中枢屏作为家庭信息中心。

---

## 一、系统架构

### 核心架构决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| Agent 框架 | OpenClaw | 提供 Gateway、Tool Call、持久记忆、多渠道等基础设施 |
| 硬件中枢 | Mac mini | 承担所有计算，24/7 运行 |
| 显示终端 | iPad 10.9"（挂墙） | 纯显示 + 触控，Safari 全屏 Web App |
| 投屏方案 | 方案 A: Web App | Mac mini 跑本地 Web 服务，iPad Safari 全屏访问 |
| 本地推理 | FunctionGemma 270M | 智能家居专用 Tool Call 模型，~500ms 延迟 |
| 云端推理 | Claude Sonnet（备用） | 复杂/多步骤任务走云端 |
| 语音硬件 | Mac mini 麦克风 / 外接 USB 麦克风阵列 | iPad 旁放置拾音设备，连回 Mac mini |

### 三通道决策架构

```
语音输入 → ASR 转文字 (~200ms)
    → 本地意图路由器 (~30ms)
        ├─ 快通道：确定性指令 → 直接执行 Tool（目标 < 500ms）
        │   例：关灯、关电视、调温度到26度
        │
        ├─ 中间通道：FunctionGemma 推理 → Tool Call（~500ms）
        │   例：卧室灯调暗一点、帮我调个舒服的温度
        │
        └─ 慢通道：云端 LLM 推理（3-8s，配合语音反馈）
            例：帮我规划明天的日程、今天适合晾衣服吗
```

## 二、FunctionGemma 270M 本地推理方案

### 为什么选择 FunctionGemma

- Google 发布的 270M 参数专用 function calling 模型
- 量化后仅需 ~288MB 内存，可在 Mac mini CPU 上运行
- 推理延迟 10-50ms（模型本身），端到端 ~500ms
- 针对智能家居场景微调后准确率可达 96%（对比基础模型 9.9%）

### 微调要求

- 需要用你自己的家电 Tool schema 构造训练数据
- 几百条高质量的家居指令样本即可
- 支持 LoRA 微调，可在 Colab 免费完成
- 使用 Unsloth 或 HuggingFace TRL 框架

### 与其他模型对比

| 模型 | 参数量 | 延迟 (CPU) | 准确率 | 适用场景 |
|------|--------|-----------|--------|---------|
| FunctionGemma | 270M | ~435ms | 0.640 (基础) / 96%+ (微调) | 快通道 Tool Call |
| Qwen3 0.6B | 600M | ~3.4s | 0.880 | 中间通道备选 |
| Qwen3 1.7B | 1.7B | ~10.7s | 0.960 | 高准确率但延迟高 |

## 三、iPad 中枢屏方案

### 技术路径

```
Mac mini
├── 本地 Web 服务 (localhost:3300)
│    ├── WebSocket 推送设备状态/环境/日程
│    ├── 摄像头流转码 (RTSP → HLS)
│    └── 中枢屏 UI (HTML/CSS/JS)
├── OpenClaw Gateway (:18789)
├── FunctionGemma 本地推理
├── 麦克风 → 语音唤醒 + ASR
└── 家电/电视 Tool 接口

iPad 10.9"（挂墙/桌面）
└── Safari 全屏 → http://mac-mini.local:3300
     └── 引导式访问锁定（常亮/禁止退出）
```

### iPad 设置要点

- 引导式访问：设置 → 辅助功能 → 引导式访问，锁定 Safari 全屏
- 常亮：引导式访问内关闭自动锁定
- 触控：Safari 内的 Web 点击/滑动事件原生支持
- 充电：iPad 保持充电连接

### 语音唤醒方案

推荐使用本地唤醒词引擎（如 Porcupine），运行在 Mac mini 上：
- 自定义唤醒词（"小管家" / "Hey Home"）
- Mac mini 外接 USB 麦克风阵列放在 iPad 旁边
- 唤醒后走 ASR → 三通道决策 → 执行 → 屏幕反馈

## 四、UI 设计

### 主屏幕 (hub-screen-v2.html)

四栏布局，信息密度最大化：

| 位置 | 模块 | 内容 |
|------|------|------|
| 左侧（跨两行） | 时钟 + 天气 | 大字时钟、天气详情（UV/风速/降雨/日落） |
| 中上 | 环境信息 | 室内温度、湿度、室外温度、AQI |
| 中上右 | 设备状态 | 6 个设备卡片，可触控切换，状态指示灯 |
| 右侧（跨两行） | 家庭日程 | 多成员日程（我/她/宝/家），明日预告 |
| 中下左 | 摄像头 | 两路实时画面（门口/客厅），Live 指示 |
| 中下右 | 场景快捷 | 离家/回家/睡眠/影院，一键触发 |
| 底部通栏 | 语音状态栏 | 唤醒提示、Gateway 状态、能耗 |

设计规格：
- 配色：跟随系统 light/dark 自动切换
- 字体：DM Sans + Noto Sans SC（中英混排）
- 语言：中英混排
- 交互：设备卡片可点击切换；场景按钮触发对话 overlay；底部栏进入系统面板

### 系统面板 (hub-system-panel.html)

从主屏底部状态栏进入，六模块布局：

| 位置 | 模块 | 内容 |
|------|------|------|
| 左上 | Gateway 网关 | 连接状态、延迟、端口、会话数、内存/CPU 进度条 |
| 中上 | LLM Models | 三通道模型状态（Router/FunctionGemma/Cloud），延迟/调用量/准确率 |
| 右上 | Nodes 节点 | 所有连接设备节点及延迟 |
| 左下（跨两列） | Tool Call Log | 实时日志，时间/Tool/输入/延迟/状态，支持筛选 |
| 右下 | Energy 能耗 | 今日用电/费用、24h 柱状图、月度统计 |

## 五、待讨论 / 下一步
